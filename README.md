A Redis-like server built from scratch with **replication, persistence, streams, and transactions**.  
Repo: https://github.com/yl0517/codecrafters-redis-go

**What’s inside**
- TCP server handling concurrent clients + commands
- Replication: handshake protocol, multi-replica propagation, ACK sync, offset tracking  
  (WAIT / INFO for coordination + availability patterns)
- Persistence: RDB parsing + key recovery
- Streams: XADD / XREAD with blocking reads + auto-generated IDs
- Transactions: MULTI / EXEC / DISCARD with queued command execution

**Why it matters**
- Demonstrates protocol-level thinking, correctness under concurrency, and systems debugging skills.
